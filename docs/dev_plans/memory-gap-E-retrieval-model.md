# Memory Gap E — Full Retrieval Model (Vector + BM25 + LLM Re-rank)

> **Status:** Not started
> **Priority:** Medium — BM25-only retrieval from memory-gap-C is sufficient for v1 alpha;
>   this gap adds semantic search quality and LLM re-ranking
> **Depends on:** memory-gap-C (service tools, BM25 base), memory-gap-D (caller token pass-through)
> **Affects:** `avix-core/src/memory_svc/search.rs`, `avix-core/src/memory_svc/tools/retrieve.rs`

---

## Problem

The `memory/retrieve` tool in memory-gap-C uses BM25-only ranking with no vector
similarity and no LLM re-rank step. The spec's retrieval pipeline is:

```
BM25 (top-K) + Vector cosine (top-K) → RRF merge → LLM re-rank → return
```

Without vectors and re-rank, retrieval quality degrades for semantic queries where
the user's words don't match the stored summary's exact words. This gap implements
the full pipeline.

---

## What Needs to Be Built

### 1. Vector index — using `llm/embed`

Vector embeddings are generated by calling `llm/embed` via IPC, using the embedding
model configured in `KernelConfig.models` (e.g., `text-embedding-3-small`).

```rust
/// Generate an embedding vector for text using the agent's assigned model.
/// Called by memory.svc on behalf of the caller (caller token pass-through).
pub async fn embed_text(
    llm_client: &LlmClient,
    text: &str,
    caller_token: &CapabilityToken,
) -> Result<Vec<f32>, AvixError> {
    let response = llm_client.call_tool("llm/embed", json!({
        "input": text,
        "_caller": caller_token.to_ipc_repr(),
    })).await?;
    // Parse the embedding vector from the response
    let embedding: Vec<f32> = serde_json::from_value(response["embedding"].clone())
        .map_err(|e| AvixError::ConfigParse(e.to_string()))?;
    Ok(embedding)
}
```

### 2. Index storage format

Vector and BM25 indexes are stored in VFS under:
```
/users/<owner>/memory/<agent>/episodic/index/vectors.idx
/users/<owner>/memory/<agent>/episodic/index/fulltext.idx
/users/<owner>/memory/<agent>/semantic/index/vectors.idx
/users/<owner>/memory/<agent>/semantic/index/fulltext.idx
```

### 3. `IndexManager` — maintains both indexes

```rust
pub struct IndexManager {
    vfs: Arc<MemFs>,
    llm_client: Arc<LlmClient>,
    embedding_model: String,  // from KernelConfig
}

impl IndexManager {
    /// Add or update a record in both indexes.
    pub async fn index_record(
        &self,
        record: &MemoryRecord,
        owner: &str,
        agent_name: &str,
        caller_token: &CapabilityToken,
    ) -> Result<(), AvixError> {
        let mem_type = match record.metadata.record_type {
            MemoryRecordType::Episodic => "episodic",
            MemoryRecordType::Semantic => "semantic",
        };

        // Generate vector
        let vector = embed_text(&self.llm_client, &record.spec.content, caller_token).await?;

        // Load or create vector index
        let vector_idx_path = format!(
            "/users/{}/memory/{}/{}/index/vectors.idx",
            owner, agent_name, mem_type
        );
        let mut idx = self.load_vector_index(&vector_idx_path).await.unwrap_or_default();
        idx.model = self.embedding_model.clone();
        // Upsert entry
        if let Some(entry) = idx.entries.iter_mut().find(|e| e.id == record.metadata.id) {
            entry.vector = vector;
            entry.updated_at = Utc::now();
        } else {
            idx.entries.push(VectorEntry {
                id: record.metadata.id.clone(),
                vector,
                updated_at: Utc::now(),
            });
        }
        self.save_vector_index(&vector_idx_path, &idx).await?;

        // Update fulltext index (BM25 IDF weights)
        let ft_idx_path = format!(
            "/users/{}/memory/{}/{}/index/fulltext.idx",
            owner, agent_name, mem_type
        );
        self.update_fulltext_index(&ft_idx_path, record).await?;

        // Update MemoryRecord.spec.index metadata
        // (rewrite the record YAML with updated index block)
        self.write_record_index_metadata(record, owner, agent_name, mem_type).await?;

        Ok(())
    }

    /// Remove a record from both indexes.
    pub async fn remove_record(&self, record_id: &str, ...) -> Result<(), AvixError> { ... }
}
```

### 4. Updated `memory/retrieve` — full pipeline

```rust
pub async fn handle(
    svc: &MemoryService,
    params: serde_json::Value,
    caller: &CallerContext,
) -> Result<serde_json::Value, AvixError> {
    let query = params["query"].as_str()...;
    let limit = ...;

    // 1. Candidate fetch (parallel)
    let (bm25_candidates, vector_candidates) = tokio::join!(
        search::bm25_candidates(&svc.vfs, query, &owner, &agent_name, CANDIDATE_K),
        search::vector_candidates(&svc.index_manager, query, &owner, &agent_name,
                                   CANDIDATE_K, caller_token),
    );

    // 2. RRF merge (k=60)
    let merged = search::rrf_merge(
        bm25_candidates.unwrap_or_default(),
        vector_candidates.unwrap_or_default(),
        60,
    );

    // Load full records for merged candidate IDs
    let candidate_records = load_records_by_ids(&svc.vfs, &merged, &owner, &agent_name).await;

    // 3. LLM re-rank (if caller has llm:inference)
    let has_llm = caller.granted_tools.contains(&"llm/complete".to_string());
    let (ranked, relevance_map) = if has_llm {
        search::llm_rerank(&svc.llm_client, query, &candidate_records, caller_token).await
            .unwrap_or_else(|_| (candidate_records.iter().collect(), HashMap::new()))
    } else {
        // No LLM — return RRF order, omit relevance field
        (candidate_records.iter().take(limit).collect(), HashMap::new())
    };

    // 4. Format response
    let records_json: Vec<_> = ranked.into_iter().take(limit).map(|r| {
        let mut obj = json!({
            "id": r.metadata.id,
            "type": r.metadata.record_type,
            "scope": "own",
            "summary": &r.spec.content[..200.min(r.spec.content.len())],
            "tags": r.metadata.tags,
            "createdAt": r.metadata.created_at,
            "pinned": r.metadata.pinned,
        });
        if let Some(rel) = relevance_map.get(&r.metadata.id) {
            obj["relevance"] = json!(rel);
        }
        obj
    }).collect();

    Ok(json!({
        "records": records_json,
        "totalCandidates": candidate_records.len(),
        "returned": records_json.len(),
    }))
}
```

### 5. `search::rrf_merge()` — Reciprocal Rank Fusion

```rust
/// Merge two ranked candidate lists using Reciprocal Rank Fusion.
/// Returns a deduplicated list of record IDs ordered by RRF score (descending).
pub fn rrf_merge(
    bm25: Vec<(String, f32)>,   // (id, score)
    vector: Vec<(String, f32)>, // (id, cosine_similarity)
    k: u32,                     // RRF parameter, typically 60
) -> Vec<String> {
    let mut rrf_scores: HashMap<String, f64> = HashMap::new();
    for (rank, (id, _)) in bm25.iter().enumerate() {
        *rrf_scores.entry(id.clone()).or_insert(0.0) += 1.0 / (k as f64 + rank as f64 + 1.0);
    }
    for (rank, (id, _)) in vector.iter().enumerate() {
        *rrf_scores.entry(id.clone()).or_insert(0.0) += 1.0 / (k as f64 + rank as f64 + 1.0);
    }
    let mut ids: Vec<_> = rrf_scores.into_iter().collect();
    ids.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap_or(std::cmp::Ordering::Equal));
    ids.into_iter().map(|(id, _)| id).collect()
}
```

### 6. `search::llm_rerank()` — model re-rank step

```rust
pub async fn llm_rerank(
    llm_client: &LlmClient,
    query: &str,
    candidates: &[MemoryRecord],
    caller_token: &CapabilityToken,
) -> Result<(Vec<&MemoryRecord>, HashMap<String, String>), AvixError> {
    if candidates.is_empty() {
        return Ok((vec![], HashMap::new()));
    }

    let candidate_text = candidates.iter().enumerate().map(|(i, r)| {
        format!("[{}] {}", r.metadata.id, &r.spec.content[..300.min(r.spec.content.len())])
    }).collect::<Vec<_>>().join("\n");

    let messages = vec![
        json!({
            "role": "user",
            "content": format!(
                "Query: {}\n\nCandidates:\n{}\n\n\
                Rank these memory records by relevance to the query. \
                Return a JSON array of objects: \
                {{\"id\": \"mem-xxx\", \"relevance\": \"brief reason\"}}. \
                Exclude records with no meaningful relevance. \
                Return only the JSON array, no other text.",
                query, candidate_text
            )
        })
    ];

    let response = llm_client.call_tool("llm/complete", json!({
        "messages": messages,
        "system": "You are a memory retrieval assistant. Rank memory records by relevance to the query. Return only a JSON array.",
        "_caller": caller_token.to_ipc_repr(),
    })).await?;

    // Parse the JSON array of {id, relevance} from response
    let text = response["content"].as_str().unwrap_or("");
    let ranked_ids: Vec<serde_json::Value> = serde_json::from_str(text)
        .map_err(|e| AvixError::ConfigParse(format!("rerank parse error: {e}")))?;

    let mut relevance_map = HashMap::new();
    let ordered: Vec<&MemoryRecord> = ranked_ids.iter()
        .filter_map(|item| {
            let id = item["id"].as_str()?;
            let relevance = item["relevance"].as_str().unwrap_or("").to_string();
            relevance_map.insert(id.to_string(), relevance);
            candidates.iter().find(|r| r.metadata.id == id)
        })
        .collect();

    Ok((ordered, relevance_map))
}
```

```rust
    idx.model != current_model
```

## Implementation Notes

- **No new crates for vector math.** Cosine similarity is `dot(a,b) / (|a| * |b|)`.
  Implement with a simple loop over `Vec<f32>`. `ndarray` is not needed for the scale
  of memory indexes expected (< 10,000 records per agent in v1).
- **Index storage format.** Use `serde_json` to serialize `VectorIndex` and
  `FulltextIndex` to JSON bytes for simplicity. MessagePack can be swapped in later if
  VFS space becomes a concern.
- **LLM re-rank prompt robustness.** The LLM response may not be valid JSON. Wrap the
  parse step in `unwrap_or_else` and fall back to RRF order on parse failure, with a
  `tracing::warn!`. Never abort retrieval due to a re-rank parse error.
- **`_caller` token pass-through.** The `CallerContext` carries the full
  `CapabilityToken`. The `llm/embed` call forwards it as `_caller` so `llm.svc` uses
  the agent's model preference and charges tokens to the agent's quota.
